# ADR-0001: 定位为自用 + 学习的单用户自托管工具

- 状态：已接受
- 日期：2026-07-09

## 背景

远程控制本地编程 CLI 的赛道已有官方与开源方案：Claude Code 官方 Web/移动端会话、Happy Coder、VibeTunnel、omnara，以及 ttyd/sshx/Tailscale+tmux 等通用组合。「PTY+WS 浏览器终端」本身无技术护城河。若定位产品化，第一问题是对现有方案的差异化；若定位自用，重复造轮子完全成立。

## 决策

按**自用 + 学习**定位：单用户自托管，不做账号体系、多租户、计费。核心诉求锚定「远程调用 claude/codex 这类 CLI」本身。产品边界：只管理 Hub 自己 spawn 的会话，不接管外部已有终端；项目代码同步不是产品功能，是部署配置。

产品化的差异化问题推迟到本人日常真实使用之后，用使用体感回答。

## 后果

- 安全模型按单用户简化（见 ADR-0003），但不等于裸奔。
- MVP 收敛到最小可日用集合（见 CONTEXT.md 一期范围）。
- 若未来转产品化，账号/中继/多租户为增量而非重写（协议与拓扑已按可插拔设计）。
