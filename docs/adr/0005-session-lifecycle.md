# ADR-0005: 会话生命周期——Hub 常驻、断线不杀、ring buffer 重放、无 tmux 层、--resume 复活

- 状态：已接受
- 日期：2026-07-09

## 背景

移动网络切换/锁屏频繁断连，「浏览器断开会话仍在跑」是本产品核心价值。容器内为 Linux，tmux 可用但非必需。容器重启会杀死全部子进程，无廉价规避手段。

## 决策

1. **Hub 常驻持有 PTY**；浏览器连接只是观察者，WS 断开不杀会话。
2. **重连恢复**：每会话维护数 MB 输出 ring buffer，attach 时重放，随后做一次 resize 抖动逼全屏 TUI 重绘收敛画面。不做服务端 headless 终端状态机（复杂度与一期不匹配）。
3. **多端同连**：输出广播给所有连接，输入不加锁，最后 resize 者决定尺寸（tmux 式，自用无争抢）。
4. **不引入 tmux 包层**：少一层 ANSI 转发、状态栏与快捷键前缀冲突面。Hub 进程稳定性靠自身 + 容器 restart policy 兜底。
5. **复活机制**：会话元数据记录 `claudeSessionId`；容器/Hub 重启后活动任务与画面丢失，对话上下文凭 `claude --resume` 一键恢复。此残余风险对自用可接受。

## 后果

- 「容器重启 = 进行中任务丢失」为已接受代价；缓冲垫是 --resume 复活对话。
- ring buffer 重放对全屏 TUI 可能有短暂闪烁，由 resize 抖动收敛。
- 若未来需要 Hub 热升级不杀会话，可再评估 tmux 层——本 ADR 届时重开。
