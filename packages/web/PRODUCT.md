# Product

## Register

product

## Users

单一用户：开发者本人（项目作者），自托管、自用。典型场景是人不在电脑前——躺在床上、通勤路上、离开工位时——用手机浏览器远程接管家里 Docker 容器内正在跑的 Claude Code / codex 会话：看输出、按几个键回复确认、必要时新开会话。手机是第一公民，桌面浏览器是次要入口。常见环境是弱光/夜间、单手持机。

## Product Purpose

ZAgent 是远程驱动本地编程 CLI 的伪终端 PWA：xterm.js 终端页 + 移动端辅助键条（Esc / Tab / Ctrl / 方向 / Shift+Tab），经 WS 信封协议连接容器内的 Hub。成功的标准是：在手机上能真正操作 Claude Code 的全屏 TUI——不断线、不丢画面、按键够得着、看得清。它是给自己用的工具，不需要说服任何人，只需要每次打开都好用。

## Brand Personality

克制、精密、隐形。三个词：quiet（安静，不抢戏）、precise（精密，每个像素有理由）、trustworthy（可靠，状态永远诚实可见）。参照 Linear 与原生终端模拟器的气质：chrome 越少越好，终端内容是绝对主角；工具应当消失在任务里。情绪目标是「掌控感」——离家千里也确信那台机器在自己手里。

## Anti-references

- **SaaS 仪表盘味**：卡片网格、渐变按钮、hero 指标、营销感组件。这是单用户工具，不是要卖的产品。
- **装饰性极客风**：CRT 扫描线、霓虹辉光、赛博朋克贴纸。终端美学靠等宽字体与克制的深色本身成立，不靠装饰。
- 任何为了「看起来有设计」而加的东西：无意义动效、装饰性图标、多余的边框和阴影。

## Design Principles

1. **终端即界面**——UI chrome 服务于 xterm 视口，永远不与之争夺空间或注意力；能收起的都收起。
2. **拇指优先**——高频操作（辅助键、会话切换）落在单手拇指热区，触控目标 ≥44px；手机排版先于桌面。
3. **状态诚实**——连接中/已附加/已断开/会话退出永远明确可见，绝不静默失败；重连与恢复是 UI 的一等公民。
4. **速度即体验**——击键回显路径上不加任何视觉延迟；动效只用于传达状态（150–250ms），不做装饰。
5. **夜间为本**——深色是默认而非主题选项；弱光下舒适，同时对比度达 WCAG AA。

## Accessibility & Inclusion

- WCAG 2.1 AA：正文对比 ≥4.5:1，大字/图形 ≥3:1（深色底上尤其注意灰字）。
- 移动端触控目标 ≥44×44px；辅助键条常驻且在拇指可达范围。
- 深色为默认，避免大面积高亮刺眼；夜间弱光场景舒适优先。
- 所有动效提供 `prefers-reduced-motion: reduce` 降级（交叉淡入或瞬时切换）。
